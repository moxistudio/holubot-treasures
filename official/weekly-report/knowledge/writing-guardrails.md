# Weekly Report Writing Guardrails (Sampled)

This knowledge note is a distilled market sample from the runtime `weekly_report` source knowledge set.

## 1) Fixed Structure

Always output four sections:

1. 本周总结
2. 已完成事项
3. 进行中事项
4. 下周计划

Prefer grouping by 客户 / 品牌 / 项目 rather than weekday chronology.

## 2) Output Profiles (One Per Run)

- `concise`: short sync, keep four sections but only key points
- `boss`: management-facing, prioritize outcomes, risks, and support needed
- `detailed`: default mode, include clear progress-risk-next action linkage

Never mix multiple profiles in one response.

## 3) Role-Focused Framing

Adjust emphasis by role clues:

- AE/AM: client progress, requirement closure, cross-team alignment
- 策划: insight framing, strategy direction, proposal convergence
- 创意: concept/KV/script iteration, client feedback, final delivery
- 媒介/投放: launch status, performance feedback, optimization, budget risk

If multiple roles appear, keep project grouping as primary.

## 4) Risk and Evidence Boundaries

- Do not fabricate facts, metrics, client feedback, or launch outcomes.
- Missing information must be marked as “待补充”.
- Risk lines should include at least two of: source, impact, coordination needed.
- “Reference sources” are writing-policy references, not proof of external truth.

## 5) Anti-Logbook Rule

- Do not output Monday-to-Friday timelines unless user explicitly asks for date replay.
- “沟通/同步/跟进/开会” are not outcomes by themselves; include result or merge/remove.
