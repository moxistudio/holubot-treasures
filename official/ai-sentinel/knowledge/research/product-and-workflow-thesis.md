# Product And Workflow Thesis

## Core positioning

AI Sentinel is not a generic AI news app. It is an intelligence control plane that continuously transforms external changes into internal action.

The important chain is:

signal collection -> structured event -> impact judgment -> content idea -> product opportunity -> internal archive

That chain matters more than endless feed breadth.

## Why this exists

The real failure mode is not missing one headline. It is missing:

- platform capability shifts
- infrastructure shifts that change product direction
- content timing windows
- early product opportunities
- consistency in judgment across weeks and months

## Output objects worth preserving

The product planning notes point to five durable object types:

- SignalEvent
- ImpactAssessment
- ContentIdea
- ProductOpportunity
- InternalMemo

Inside the Treasure, EventItem is the long-lived memory unit. But these other objects remain useful operational views around the same core event.

## Product surfaces

The notes suggest five useful surfaces:

- Daily Brief
- Radar Board
- Content Studio
- Opportunity Map
- Private Archive

The Treasure package does not need to render all of them itself, but its output contracts should remain compatible with them.

## Source hierarchy in product terms

The product notes used L0/L1/L2 language. DR-4 used Tier 1-4 language. The practical merge is:

- Tier 1 roughly matches L0 official first-party anchors
- Tier 2 covers authority media and analyst expansion
- Tier 3 and Tier 4 cover early ecosystem monitoring and noise-prone social layers

## Public vs internal split

One of the most durable ideas in the current archive is the split between:

- public-safe brief output
- internal-only route notes

That split should remain a first-class pattern because it lets AI Sentinel create reusable external value without leaking internal project leverage.

## MVP principle

Build the shortest path that proves the whole chain works:

- collect signals
- rank them
- write a brief
- create reusable memory
- generate content and opportunity implications

Do not optimize for app breadth before this chain works reliably.
